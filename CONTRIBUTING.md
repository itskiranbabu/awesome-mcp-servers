# Contributing to Awesome MCP Servers

First off, thank you for considering contributing to Awesome MCP Servers! It's people like you that make this resource valuable for the entire AI and developer community.

## üåü How Can I Contribute?

### Reporting Issues

- **Check existing issues** before creating a new one
- Use a **clear and descriptive title**
- Provide **detailed information** about the issue
- Include **steps to reproduce** if applicable

### Suggesting MCP Servers

We love new suggestions! Here's how to suggest an MCP server:

1. **Check if it already exists** in the README
2. **Verify the server is active** and maintained
3. **Test the server** yourself before suggesting
4. **Provide complete information**:
   - Name and GitHub link
   - Clear description
   - Language/framework
   - Why it's valuable
   - Category it belongs to

### Pull Request Process

1. **Fork the repository**
2. **Create a new branch** (`git checkout -b feature/add-awesome-server`)
3. **Make your changes**
4. **Test all links** to ensure they work
5. **Follow the formatting guidelines** (see below)
6. **Commit with clear messages** (`git commit -m 'Add: Awesome MCP Server'`)
7. **Push to your fork** (`git push origin feature/add-awesome-server`)
8. **Open a Pull Request**

## üìã Guidelines

### Quality Standards

MCP servers and resources should be:

- **Active**: Recently updated or maintained
- **Relevant**: Related to MCP or AI/ML learning
- **Accessible**: Publicly available (open-source preferred)
- **Valuable**: Provides clear benefit to developers
- **Documented**: Has clear documentation or instructions

### Formatting Guidelines

#### Adding an MCP Server

```markdown
- [**server-name**](https://github.com/user/repo) - Brief, clear description
```

#### Adding a Learning Resource

```markdown
- **[Resource Name](https://resource-url.com/)** - Description, provider, duration, cost
```

#### Adding a Tutorial

```markdown
- **[Tutorial Title](https://tutorial-url.com/)** - Brief summary of what it teaches
```

### Description Guidelines

- **Be concise**: 1-2 sentences maximum
- **Be specific**: What makes it unique?
- **Be objective**: Avoid marketing language
- **Be accurate**: Test before submitting

### Categories

Place resources in the appropriate category:

- **Official MCP Servers**: Anthropic-maintained servers
- **Community MCP Servers**: Third-party servers
- **MCP Clients**: Applications that use MCP
- **Development Tools**: Tools for building MCP servers
- **AI Learning Resources**: Courses, tutorials, books
- **Tutorials & Guides**: How-to content
- **Best Practices**: Guidelines and patterns
- **Use Cases**: Real-world examples

## üö´ What NOT to Submit

- **Broken links** or inactive projects
- **Duplicate entries** already in the list
- **Self-promotion** without clear value
- **Paid-only resources** without free tier or trial
- **Unrelated resources** not about MCP or AI/ML
- **Low-quality** or unmaintained projects
- **Spam or malicious** content

If you think a new category would be valuable:
